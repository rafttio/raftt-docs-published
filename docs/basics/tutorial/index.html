Tutorial Using our Sample To-Do App

We created TooDoo, a simple to-do application, as a sample project to help you quickly onboard to Raftt, even without using your own code.

The project is written mainly in Python (Flask) with some Go, but familiarity with these languages isn't necessary to successfully complete this tutorial.

Your First raftt upâ€‹

In this section, we'll get you up and running with your very own Raftt environment and explain what happens under the hood. Take the following steps -

  1. Clone the project's repo - git clone https://github.com/rafttio/toodoo
  2. Run raftt up and wait for Environment spawned successfully! to be printed
    1. You will be prompted to log in with your GitHub account
  3. Run raftt status to see the status of all the environment services.

Under the Hoodâ€‹

When you ran raftt up, you started a sequence of events -

  1. Authentication with GitHub
  2. Raftt's configuration file, raftt.yml, is detected by the Raftt client.
    1. In this case, raftt.yml was cloned with the repo. If it doesn't exist, it is automatically generated with default values.
  3. The raftt.yml file refers to the environment definitions - the Docker Compose file and the dev container definitions.
  4. A new private and isolated environment is spawned in Raftt's cloud.

If you have docker installed, you can run docker ps and see that nothing runs on your machine, and it's all remote.

Working With the Envâ€‹

Now that your env is up, you can start working on the project and modify the code.

Changing the Color of a Buttonâ€‹

In line 74 in the file <repo>/templates/index.html, change btn-primary to btn-danger, save, and reload the page. The submit button should now be colored red.

Breaking the Codeâ€‹

Now let's see what happens if you make a mistake.
Add a change that will cause the main process to crash.
You can go to <repo>/app.py and add This won't work to line 6.
Since it is python code, you need to restart the web container main process for the change to have an effect.
Do this by running raftt restart web.
Note that it only restarts the main process, and the container itself isn't re-deployed.
If you now refresh the webpage, you'll see the page isn't loading.
To debug the issue, you should run raftt logs web and see the error in the container logs.
(AS - add screenshot)
To get the service back up, revert all changes using git checkout . and run raftt restart web.
You can now see the app is working again, and you can also recheck the logs to see the process started successfully.

Debugging the Applicationâ€‹

One of the most common things devs want to do is debugging their environment.
Raftt currently supports interactively debugging Python and Go code using Visual Studio Code or any JetBrains IDE. Additional support is continuously added.

  • VS Code
  • JetBrains

To debug the application, follow these steps -

  1. Open the project in your VS Code
  2. Add a breakpoint inside the function create in the file app.py
  3. Start debugging using the debug configuration Raftt
    1. Debugging in this project is pre-configured by files committed in the repo. See here for details on how to do it yourself
  4. Create a task in the TooDoo
  5. See it stopped where you put the breakpoint
  6. Debug the remote service as if it were local

Branch Switchingâ€‹

Raftt seamlessly integrates with your workflow and makes switching between branches a breeze.
Switching to a new branch spawned a new environment while the previous environment(s) are waiting if you want to come back to them later.

Database Changesâ€‹

Switch to a different branch using git checkout emoji and refresh the page.
You can now see our new feature - choosing an emoji for each task. Note the tasks changed, both in content and in database schema (to support the emoji), but with Raftt, this change is seamless.
You can go back to main using git checkout main and see any changes to the task list remain.

Adding Servicesâ€‹

Switch to a different branch using git checkout active-users and refresh the page.
We added a counter that shows the number of users connected to our awesome TooDoo app. If you run raftt status, you'll notice a few changes -
