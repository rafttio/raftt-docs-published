SSSSSSSSSSSpawn Your First Environment in 5 Minutes Using Raftt

You can have a fully managed dev env set up in less than 5 minutes if you use docker-compose and your repo is on Github or Bitbucket. If you want to play around with a sample project, check out our tutorial.

Step 1 - Install Raftt‚Äã

  • macOS
  • Linux
  • Windows

Use Homebrew or cURL to install Raftt CLI on macOS.

# Use Homebrew
brew install rafttio/tap/raftt  

# Or use cURL
curl -fsSL get.raftt.io/install | bash

Use Snapd or cURL to install Raftt CLI on Linux.

# Use Snapd
sudo snap install raftt --classic

# Or use cURL
curl -fsSL get.raftt.io/install | bash
  1. Download Raftt portable binary (x64)
  2. Add its location to the PATH
  3. Install Raftt service by running raftt daemon install
Validate

To verify the installation was successful, run raftt version in your shell.

Step 2 - Spawn the Environment‚Äã

You are welcome to try our tutorial to get a quick feel for Raftt's usability :)
If you want to try it with your project, run raftt up from inside the repository to spawn your environment.

Note

The repository you are using must have a docker-compose file. You can check our tutorial for an example or see our prerequisites for more info.

You can run multiple environments concurrently, so don't be afraid to try it out.

The up command does the following things:

  1. Create Raftt's configuration file - raftt.yml, with reference to your docker-compose file. You can edit this if you need to.
  2. Initialize your authentication to Raftt through OAuth via Github or Bitbucket (GitLab is coming soon!).
  3. Spawn a remote environment according to your docker-compose file with the addition of the dev container.

Step 3 - Success‚Äã

You can now start accessing your environment -

  1. Open your browser to one of your published ports localhost:<PORT>
    1. Make changes to code and see them reflected remotely!
  2. Switch between git branches and see your remote environment change seamlessly.
  3. raftt sh dev for accessing your dev container.
  4. raftt logs SERVICE for seeing logs.

Read the CLI reference for more commands.
