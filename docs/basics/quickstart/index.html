Spawn Your First Environment in 5 Minutes Using Raftt

If you are using docker-compose and your repo is on Github or Bitbucket, you can have a fully managed developer environment up in less than 5 minutes! If you want to play around with a sample project, see our tutorial.

Step 1 - Install Rafttâ€‹

  • macOS
  • Linux
  • Windows

Using Homebrew, run:

brew install rafttio/tap/raftt

Option 1 - Use Snapdâ€‹

Using snapd, run:

sudo snap install raftt

Option 2 - Use cURLâ€‹

Using cURL, run:

curl -fsSL <installation_script_url> | bash

Option 1 - Use Scoopâ€‹

Using Scoop, run:

scoop bucket add raftt <relevant_url>
scoop install raftt

Option 2 - Download a Portable Executableâ€‹

  1. Download Raftt portable binary (x64)
  2. Add its location to the PATH
  3. Install Raftt service by running raftt daemon install from a terminal with administrator privileges
Validate

To make sure the installation was successful, run raftt version

Step 2 - Initial setupâ€‹

To spawn your environment, run raftt up from inside the repository.
You are welcome to try our tutorial as you

Note

It must be a repository with a Docker Compose file that only refers to a single repo. You can use our example repo.

The up command will do the following things:

  1. Create Raftt's configuration file - raftt.yml, with a reference to your docker-compose file. If needed, you may edit this.
  2. Initialize your authentication to Raftt through OAuth via Github or Bitbucket (GitLab coming soon!)
  3. Spawn a remote environment according to your docker-compose file with the addition of the dev container.

Step 3 - Successâ€‹

You can now start accessing your environment -

  1. Open your browser to one of your published ports localhost:<PORT>
    1. Make some changes to code and see it reflected remotely!
  2. Switch between git branches and see your remote environment change seamlessly
  3. raftt sh dev for accessing your dev container
  4. raftt logs SERVICE for seeing logs

See the CLI reference for more commands.
