Prerequisites for Using Raftt

Raftt is still in beta, so it doesn't yet support everything we want it to.
If you want to use Raftt but these prerequisites limit your ability to do so, join our community or contact us, and we'll see what we can do for you.

Environment Definition Toolâ€‹

Raftt currently supports environments defined using Docker Compose. We'll soon add support for additional definition methods, such as Helm Charts, raw Kubernetes manifests, Terraform, etc.

Single Repo Environmentsâ€‹

To successfully spawn your environment, Raftt requires the whole env to be a part of a single repo.

Repository Hosting Serviceâ€‹

Raftt requires OAuth authorization with your repo hosting service to get access to the repo and seamlessly follow your workflow. Raftt currently supports GitHub and BitBucket. Support for GitLab is coming soon.

Prerequisites for Debugging with Rafttâ€‹

One of Raftt's main benefits is allowing devs to use their local IDE to debug, even when the containers are running remotely. Even though Raftt is agnostic to your tech stack when spawning environments, debugging capabilities are supported for specific programming languages and IDEs. See further details here.
