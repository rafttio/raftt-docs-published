Raftt Configuration File - raftt.yml

The configuration of Raftt is located in the raftt.yml file that is created the first time you run raftt up for the repository. (It can also be created by running raftt setup before running raftt up). Once the file is created, you need to modify it according to your needs, and we recommend committing it to your repo.

See below a sample raftt.yml file containing all possible attributes.
A more detailed explanation can be found [below](raftt.yml Reference)

    build:
        dockerComposeFile: ./docker/docker-compose.yml  # The path of of the docker-compose file used to 
                                                        # define the environment
        workdir: ./docker/                              # The working dir when spawning the environment
    devContainer: dev_container/dev-compose.yml         # The path of the dev container that XYZ
    admiralHost: admiral.public.raftt.io                        
    tearDownScript: curl admiral:8080/downall
    secrets:
        aws-accesskeyid-beta:
            inputcommand: 
            outputenv: AWS_ACCESS_KEY_ID
        ... 
    data:                                               # Database seeding configuration
    - service: admiraldb
        type: postgres
        user: postgres
        dump: dev_container/dump.sql
    - service: backenddb
        type: postgres
        user: postgres
        dump: dev_container/backenddump.sql
    warmUpScript: dev_container/warmup.sh

Each dictionary entry contains the following attributes:

    tip

    Since this command runs locally, we recommend using an OS-independent command, so the same raftt.yml file can be shared between team members working with different operating systems.
    A possible way to do it is having this command run a short OS-independent python code whose output is the secret.

  • outputenv - The name of the Kubernetes Secret that will contain the output of inputcommand.
