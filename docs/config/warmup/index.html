Warmup Script

Note

The warmup script is optional. In most cases, environments will be spawned successfully without it.

The warmup script is executed from the context of the dev container before building and deploying the rest of the environment.
This script can help you prepare whatever needs to be prepared for the containers build to be successful, or for preparing the dev container for convenient use.

Remember

Add all warmup script dependencies to the dev container using its Dockerfile.

Here are a few examples of what can be done in the warmup script

Example 1 - Download Build Dependenciesâ€‹

Let's assume your services are written in go, and you compile them on your dev container. It may be convenient to have all the go modules odwnloaded to the dev container so compiling will be speedy. The following warmup script is an example for how to accomplish that:

cd /code
nohup go mod download > /dev/null & # Perform async so the warmup script does not block build and deploy process.

Example 2 - Compiling Protobufâ€‹

Your services may communicate between them using protobuf, GraphQL, or other schema language that requires generation. You can automatically generate the relevant files using something like:

find . -name '*.proto' -exec protoc --go_out=paths=source_relative:./ '{}' ';'
