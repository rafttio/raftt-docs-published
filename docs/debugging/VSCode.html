Debug Remote Environments Using Visual Studio Code

Microsoft's VS Code is the most popular IDE amongst software developers.
Using Raftt allows devs to continue working and debugging in their favorite IDE with their own setup and configuration while enjoying all the benefits of an environment running remotely.

Note

Raftt currently supports debugging Python and Go code. Join our Slack community or contact us to enquire about upcoming support for additional languages.

Configuration​

To configure VS Code to debug with Raftt, you must modify the debug configuration. This configuration differs a bit between Go and Python.

  • Python
  • Go

Add the following attributes to the debug configuration in the project's launch.json file.

Note

Keeping the JSON file platform-independent makes it easier to share between team members that might use different operating systems.

"windows": {
    "debugAdapterPath": "${env:HOMEPATH}/.raftt/plugins/debugpy/adapter",
},
"linux": {
    "debugAdapterPath": "${env:HOME}/.raftt/plugins/debugpy/adapter",
},
"osx": {
    "debugAdapterPath": "${env:HOME}/.raftt/plugins/debugpy/adapter",
},
"serviceName": <SERVICE_TO_DEBUG>

In the IDE​

Modify the debug configuration file - launch.json, and the workspace settings file settings.json.

Debug Configuration​

Add the following attribute to the relevant debug configuration in the launch.json file:

"env": {
    "RAFTT_SERVICE": <SERVICE_TO_DEBUG>,
    "WORKDIR": "${workspaceFolder}"
}

Make sure that the value of the program points to the program you wish to debug.

Workspace Settings​

Add the following attribute to the settings.json file (the one in the .vscode folder in the workspace directory)

Note

Keeping the JSON file platform-independent makes sharing between team members who might use different operating systems easier.

{
        "go.alternateTools": {
            "dlv": "~/.raftt/plugins/dlv/dlv",
        },
}

It states the path of Raftt's debug adapter, and yes - '~' works for Windows!

caution

The changes to settings.json affect the whole workspace and not just specific debug configuration.
If you wish to debug without the help of Raftt, you need to empty or remove the file.

Debugging​

Once the plugin is installed and configured, you can debug the service just like you would if it ran locally, e.g., add breakpoints, view symbol, and more.
