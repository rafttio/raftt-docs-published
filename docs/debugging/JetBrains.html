Debug Remote Environments Using Jetbrains IDEs

JetBrains IDEs are some of the most popular IDEs amongst software developers.
Using Raftt allows devs to continue working and debugging on their favorite IDE with their customized setup and configuration while enjoying all the benefits of a remotely running environment.

Note

Raftt currently supports debugging Python and Go code. Join our Slack community or contact us to enquire about upcoming support for additional languages.

Installing the plugin​

Install the plugin directly from JetBrains Marketplace. It is compatible with all IntelliJ-based IDEs!

This configuration differs between different languages.

  • Go
  • Node.js

General Configuration​

To debug Node.js projects, you must modify your npm run configuration with the following changes:

  1. Change the node interpreter
    1. On Linux/MacOS - ~/.raftt/plugins/node/jetbrains/node
    2. On Windows - ~/.raftt/plugins/node/jetbrains/node.exe (yes, '/'s also work for Windows!)
  2. Change the package manager
    1. If npm is used - ~/.raftt/plugins/node/jetbrains/npm
    2. If Yarn is used - ~/.raftt/plugins/node/jetbrains/yarn
  3. Add environment variable RAFTT_SERVICE=<desired-service>

TypeScript-specific Configuration​

To debug TypeScript projects, add inlineSourceMap: true to your tsconfig.json file.
The sourceMap attribute must be removed from tsconfig.json, if it exists.

Recompiling the Code​

If you wish Raftt to recompile the code in every run or debug operation, configure a before launch task with a configuration that matches the one of the project:

  1. Create a before launch task of type Run npm script.
  2. Select the proper package.json that should match the one you configured above.
  3. Select run as the command.
  4. Select the script that builds the project. It's usually called build or build:<env>.
    Note: this script must save its build result to the path used by the start script used in the run/debug configuration.
  5. Node interpreter
    1. On Linux/MacOS - ~/.raftt/plugins/node/jetbrains/node
    2. On Windows - ~/.raftt/plugins/node/jetbrains/node.exe (yes, '/'s also work for Windows!)
  6. Select the proper package manager that should match the one you configured above.
  7. Add environment variable RAFTT_SERVICE=<desired-service>.

Debugging​

Once the plugin is installed and configured, you can debug the service just like you would if it ran locally, e.g., add breakpoints, view symbols, and more.
