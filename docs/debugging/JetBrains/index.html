Debug Remote Environments Using Jetbrains IDEs

JetBrains IDEs are some of the most popular IDEs amongst software developers.
Using Raftt allows devs to continue working and debugging in their favorite IDE with their own setup and configuration while enjoying all the benefits of an environment running remotely.

Note

Raftt currently supports debugging Python and Go code. Join our slack community or contact us to enquire about upcoming support for additional languages.

Installing the pluginâ€‹

Install the plugin as described here. These instructions are from the IntelliJ IDEA docs, bit they are the same for other JetBrains IDEs.
The plugin's JAR file can be found in the following path -

  • macOS
  • Linux
  • Windows
~/.raftt/plugins/RafttLauncher/RafttLauncher-<version>.jar
~/.raftt/plugins/RafttLauncher/RafttLauncher-<version>.jar
%USERPROFILE%\.raftt\plugins\RafttLauncher\RafttLauncher-<version>.jar

Configurationâ€‹

The only required configuration is adding an environment variable RAFTT_SERVICE stating which of the services is the one you want to debug. The easiest way to do that is stating it under Environment in the debug configuration you plan on using.

Once the plugin is installed and configured, you can debug the service just like you would if it ran locally - add breakpoints, view symbols, etc.
