Git Workflow Integration

One of Raftt's unique benefits is integration with your Git workflow.
Raftt lets you switch between branches without worrying about the state of your dev environment.
When leaving a branch while having an active environment, the whole env is "frozen" for future use, while the new branch enjoys a clean environment.

We'll describe what happens when users switch to a different branch using git checkout. The details differ between two cases depending wether the checked out branch has a "frozen" environment.

  1. A new branch - the newly checked out branch doesn't have a "frozen" environment.
  2. A frozen branch - the newly checked out branch already had an active env that was frozen by checking out another branch.

A New Branchâ€‹

  1. The code is checked out by the user in the local machine.
  2. Raftt recognizes it using githooks and notifies the backend.
  3. The backend "freezes" the running environment
    1. The containers state is backed up
    2. The storage is backed up
    3. The containers are taken down
  4. The container starts a new environment for the new branch, similarly to what is describe in how raftt works?
Note

The freezing of the environment happens in the background, so you can quickly start working on the branch you just checked out

A Frozen Branchâ€‹

  1. The code is checked out by the user in the local machine.
  2. Raftt recognizes it using githooks and notifies the backend.
  3. The backend "freezes" the running environment
    1. The containers state is backed up
    2. The storage is backed up
    3. The containers are taken down.
  4. The backend recognizes the new branch has a "frozen" environment
  5. The Frozen environment is woken up
    1. A new environment controller is build
    2. All containers are loaded
    3. The containers state is restored from backup
    4. The storage is restored from backup
Note

The freezing of the environment happens in the background, so you can quickly start working on the branch you just checked out
