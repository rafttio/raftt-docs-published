File Syncing

The dev container's role in the environment is to mimic the dev machine.
To do that, it's important to keep the local repo folder in sync with the corresponding folder in the dev container.

This is done by constantly syncing using an OSS tool called Syncthing.
The whole folder is synced, excluding what is in .gitignore files.
The file sync lets you develop locally and have it affect the remote environment.

Thanks to this file syncing mechanism, you can work on your local machine, with your favorite IDE and tools, and run the code remotely.
